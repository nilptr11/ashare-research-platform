# 推理边界

## 基本原则

1. 先确认数据日期、来源、质量和缺口，再写结论。
2. 区分事实、推断和假设。
3. 使用最小必要数据路径，避免为了套模板读取无关数据。
4. 任何缺口都要说明影响：`block`、`degrade` 或 `warn`。
5. 不输出交易执行指令。

## 来源优先级

| 事实类型 | 优先来源 |
| --- | --- |
| 行情、估值、资金、公告、财务 | mart |
| 筛查、排序、聚合信号 | feature |
| 产业外部事实 | evidence |
| 公司、产品、客户、产业链关系 | accepted knowledge |
| 分析过程和输出留痕 | runs，仅作留痕 |

## Feature 使用边界

Feature 可以支持：

- 候选发现；
- 强弱排序；
- 市场认可度初筛；
- 交叉验证入口。

Feature 不能单独支持：

- 公司正宗程度；
- 订单真实兑现；
- 基本面证伪；
- 买卖或仓位动作。

## 公司业务暴露度

判断公司处于产业链某环节时，至少需要一种可追溯支持：

- 公司公告、年报、半年报、招股书、定期报告；
- 公司 IR、互动平台、交易所问询回复；
- 合格 evidence；
- accepted knowledge。

只有概念成分、热榜、人气榜、涨停池时，只能写“市场关注线索”，不能写“业务暴露已验证”。

## 降级规则

| 缺口 | 写法 |
| --- | --- |
| 缺市场数据 | 阻断市场强弱结论 |
| 缺 feature meta 或 quality 非 ok | 只能使用 mart 明细或写降级 |
| 缺外部产业证据 | 产业兑现结论降级 |
| 缺公司披露或 knowledge | 公司正宗程度降级 |
| 缺实时数据 | 只能基于最近完整交易日 |

## 禁止事项

- 不使用 raw store 作为默认事实源。
- 不把 run/report 当事实源。
- 不用外部搜索覆盖 mart 中已有事实。
- 不输出买入、卖出、加仓、减仓、满仓、单吊、止盈止损、下单等交易执行建议。
